# Information-architecture review — 2026-07-21

Reviewed the flow and placement of data across the app's nav/tabs/cards: does each
component sit on the surface its arriving user expects, where does the credibility
rating belong, what should season/week/scope controls default to in-season vs
offseason, and what should the app lead with. Prompted by Doug's read that "the elo
only shows up on the career page for the league standings, but then the awards are
hidden — important info split across two pages, not intuitive."

- **HEAD at review time:** `a2550f2` (master).
- **Method:** code-based (no screenshots — below-the-fold claims are code-derived
  estimates). One inventory subagent per screen (`/league`, `/stats`, `/picks`,
  `/market`, `/recap`+`/wrapped`, nav shell) recording every block's render condition
  and every control's default rule, plus a design-law extraction pass
  (`DESIGN.md`, ADR-0018/0030/0032/0035, `PRODUCT.md`). Three analyses (placement
  architecture, temporal defaults, user-focus model) were each adversarially
  verified, then a completeness critic reconciled their conflicts. All findings below
  cite verified code.
- **Not covered:** `/settings` and `/how-to-play` content (deliberately shallow);
  visual/pixel critique (that's `design-review`'s job, and would need the capture
  harness).

## The diagnosis in one paragraph

`/league`'s Standings tab is one `{#if scope === 'alltime'} … {:else}` branch: the
`RatingLadder` renders only on the All-time side, `LeagueHonors` only on the season
side — they are mutually exclusive **by code structure**, and the scope select is the
only door between them. `+page.server.ts:73-74` then defaults a bare offseason visit
to `alltime`, so for the seven months when last season's honors _are_ the league's
content, the default paint is a career W-L table plus a slow-moving rating ladder,
with the champion one undiscoverable dropdown flip away. The same
`isSeasonInProgress` flip silently swaps `/stats` to Career (materializing
RatingBand) and `/market` to Pooled (landing on a "pick a season" dead-end card),
while `/recap` — the honors archive — anchors to the _calendar_ season (2026, zero
graded weeks) and opens empty. Five surfaces answer the same question ("what
timeframe when nothing is live?") five different ways, even though
`resolveSeasonYear` already computes the right anchor and three of them already call
it — then override it.

## Ranked findings

### P1 — confusing today

1. **RatingLadder and LeagueHonors are mutually exclusive by construction.**
   `src/routes/(app)/league/+page.svelte` — ladder only inside the `alltime` branch
   (~line 392), honors only inside the `season` branch (~line 454), one `scope`
   state. The tab can never show both. This is the reported pain, verbatim, as a
   code shape.
2. **The offseason default actively hides the honors.**
   `league/+page.server.ts:73-74` — `defaultScope = seasonParam != null ||
latestSeasonInProgress ? 'season' : 'alltime'`. A bare July visit lands on
   All-time + ladder; champion/spoon/titles invisible in exactly the window they
   peak.
3. **`/recap` is dead on arrival in the offseason.** `recap/+page.server.ts:13` uses
   `getCurrentSeasonYear()` (2026, no graded weeks) instead of `resolveSeasonYear`,
   honors no `?season=`, and has no picker — 2025's full SeasonShelf + hardware +
   recaps sit unreachable while the page renders its empty state. Recap pushes
   deep-link here, so it's also many users' PWA resume point.
4. **The rating is split across two pages with _opposite_ scope gates and no
   cross-link.** RatingBand renders only at `/stats` Career scope
   (`StatsHero.svelte:47-49`); the ladder only at `/league` All-time scope. In
   season — when defaults are `season` on both pages — **both** rating surfaces are
   hidden behind non-obvious flips, and `RatingBand`'s "#N in league" line has no
   path to the ladder that explains it. (There is no `?scope=` URL param at all;
   `/league?season=alltime` parses as a season and lands on the branch where the
   ladder can't render.)
5. **`/market`'s offseason default is a dead end.** `market/+page.server.ts:37`
   defaults to `pooled`; the slice chips default to `teams`; pooled has no per-team
   roster by design — so the default detail region is a card telling the user to
   change a control they never touched.

### P2 — friction

6. **Five temporal-default idioms for one question.** Census of every control:
   `/league` scope → `alltime` offseason; `/stats` scope → `career`; `/market` scope
   → `pooled`; `/recap` → current calendar season; `/wrapped` → latest _generated_
   Wrapped (the one model citizen). `resolveSeasonYear`
   (`src/lib/server/seasonDefault.ts`) already computes the right anchor and
   league/stats/market all call it — then throw it away when
   `isSeasonInProgress` is false.
7. **In-season, honors are the 4th stacked block** on the tab the product audit
   already flagged as the systemic density risk — below the full table and the race
   chart. Answer-first (DESIGN.md P4) inverted for the layer Doug values most.
8. **RatingBand appears/disappears on a calendar boundary** on `/stats` with no cue,
   and its "#N in league" rank is a standings fact carved into the analytics page
   (ADR-0018 strain — `StatsHero`'s own comment disclaims standings tiles).
9. **Week-1 edge window:** `availableSeasons` comes from
   `leaderboard_season_totals`, so until week 1 _grades_, `isSeasonInProgress` is
   evaluated against 2025 and no "This season · 2026" pin exists anywhere — the live
   season isn't even an explicit option between schedule-seed and first grade.
10. **`/picks` puts two spectator widgets** (`PicksStatusBoard`,
    `AllInDeclarations`) between the arriving picker and their unpicked game cards —
    the highest-frequency arrival (pick-reminder push) wants "what do I still need
    to pick" first (DESIGN.md P6).

### P3 — polish

11. **`/market` WeekSlate ignores the page's own season control** (always tracks the
    current season's next slate; renders its empty state above whatever the dropdown
    selected). Should fold into #692's reshape — verify #692's scope covers it.
12. **WrappedPromo floats above the Tabs** regardless of tab/scope — season-specific
    messaging over an All-time table, one more block on the dense surface.
13. **RecapFlash and WrappedFlash are two shell-level interrupts** for the same
    season-story moment, with separate seen-tables. Needs one named owner.

## What users are most interested in (the moment model)

Moments of the week for a 6-friend pick'em, and what the arriving user wants first:
**Tue–Wed** (recap digested): what happened — recap prose, hardware, movement.
**Thu–Sun morning** (highest-frequency arrival, via pick-reminder push): what do I
still need to pick. **Sunday live:** am I covering — the sweat board.
**Sun night–Mon:** did I move — standings + arrows, then the trash-talk payload.
**Offseason (7 months):** the season's story — champion, spoon, titles, Wrapped,
career bragging rights. There is no other content.

The derived interest ranking:

1. **Results & standings movement** — wanted in 3 of 5 moments; the score of the
   meta-game.
2. **Trash-talk material** (recap, hardware, who-hasn't-picked, bad beats) — the
   reason the app exists, but it rides _on_ results.
3. **Making picks** — highest frequency, purely instrumental: fast, not featured.
4. **Live sweat** — intense, Sunday-only.
5. **Season honors** — episodic: near-zero midweek, **dominant** at season end and
   for the entire offseason. Their rank is _seasonal_, which is why a static block
   order can't serve them.
6. **The credibility rating** — a slow-arc identity stat checked monthly; never the
   reason anyone opened the app.
7. **Market/situational analytics** — an occasional browse for the league's sharps.

The current offseason `/league` default is this ranking exactly inverted: it leads
with #6 (ladder) and an aggregate table, and hides #5 at its seasonal peak.

**The focus answer: scoreboard-first, honors-lit.** Every arrival answers "who's
winning / did it change" first; in the offseason, the completed season's honors ARE
that answer. The rating is context, not focus — it belongs _beside_ the standings,
always visible, never leading. (A conversation-first posture — calendar-driven
defaults chasing the chat moment — was evaluated and rejected: it makes the same tap
land somewhere different on Wednesday than Monday, systematizing the default
confusion, and the app feeds the chat, it doesn't host it.)

## The reconciled plan

Three analyses each survived adversarial review; their conflicts were reconciled as
follows into one wave plus one contingent phase.

### Rule 1 — temporal defaults

> **Every timeframe control defaults to the most recently _graded_ thing.
> Aggregate scopes (All-time, Career, Pooled) are always an explicit choice, never
> a default.**

In-season the anchor is the live season; offseason it is the last completed season.
This supersedes **#638's offseason half** (default-to-aggregate) while keeping its
pin-honesty half: a concluded season is pinned **"Last season · YYYY"**, never "This
season", and is deduped out of the Past-seasons optgroup. The pin's availability
check runs against `Math.max(currentSeasonYear, seasonYear, ...availableSeasons)` so
"This season · 2026" is _offered_ from schedule-seed while the _default_ stays 2025
until week 1 grades (no empty-standings default, by construction — fixes finding 9).

### Rule 2 — ordering

> **A crowned season leads with its crown.** When the _viewed_ season is concluded,
> honors lead (honors → table → ladder); while it's in progress, the table leads
> (table → honors → ladder). Race chart last (or behind a one-tap disclosure).

Data-keyed (`!isSeasonInProgress(viewedSeasonYear)`), not calendar-keyed — the same
season page renders identically in July and November, and browsing 2024 mid-2026
is also honors-first. This is what makes honors prominent without a static reorder
that would demote in-season standings movement (the #1 want).

### Rule 3 — where the ELO lives

- **The room's ladder: `/league` Standings tab, always on**, beneath the table in
  _both_ scopes, with mandatory grain copy ("Career rating — market, not points").
  This supersedes the #637 same-grain comment at `league/+page.svelte:374-378`
  (record it in the ADR-0032 margin). Never a column in the standings table —
  ADR-0032 §9's conflation ban stands (and the #631 no-fourth-column comment).
- **Your own rating: `/stats` Career hero stays the canonical home** (ADR-0032), and
  the _season_ hero gains a compact, explicitly-labeled "Career rating" chip
  (number + tier + #N) so the rating is default-visible **year-round** — strictly
  better than today, where it's default-hidden for the whole season. The "#N in
  league" line links to the ladder.
- **New `?scope=alltime` URL param**, honored in the server's `defaultScope`
  computation — the only way the cross-link can work, and it makes the All-time +
  ladder view shareable for the first time.

### Wave 1 — the change list (small, mostly deletion-of-exceptions)

1. `/league` `defaultScope` → season year-round; add `?scope=alltime`.
2. Hoist `RatingLadder` always-on with grain copy; apply Rule 2 ordering; race chart
   last or one-tap disclosure.
3. **Retire WrappedPromo** — its CTA folds into `LeagueHonors`' existing `/wrapped`
   link. (The dense tab nets _lighter_; displacement corollary paid.)
4. `/recap`: `resolveSeasonYear` + minimal season picker (reuse the
   `seasonScopeOptions` pattern, no all-time option) + season-qualified week anchors
   in push links and `WeeklyHardware`'s `recapHref`.
5. `/stats` `defaultScope` → season year-round + the career-rating chip on the
   season hero (note as an ADR-0032 §9 surfacing amendment) + ladder link.
6. `/market` `defaultScope` → season; keep Pooled pinned; slice fallback
   pooled+teams → favorites; confirm #692 covers the WeekSlate-ignores-the-control
   finding, extend it if not.
7. `/picks`: unpicked game cards above `PicksStatusBoard` + `AllInDeclarations`.
   (Safe: once you've picked everything, `upcoming` is empty and the status board
   effectively leads anyway.)
8. `seasonScopeOptions`: offseason "Last season · YYYY" pin + dedupe; pin gate vs
   `max(currentSeasonYear, …)`.
9. Same-PR bookkeeping: record Rules 1–3 in `DESIGN.md`'s vocabulary; margin notes
   on ADR-0032 (chip, ladder ungating) and #638 (offseason half superseded); update
   the all-time e2e spec (referenced at `league/+page.svelte:179`), offseason-default
   e2e assertions, and `stats.ts` `seasonScopeOptions` unit tests; run
   `refresh-demo-snapshot` after the `/league` reorder (the demo mirrors it and the
   drift-guard won't catch it); sanity-check `/how-to-play` copy (likely unaffected —
   no feature added or removed).

### Phase 2 — contingent: the Trophy Room

The placement analysis's recommended destination: `/league` becomes
**Standings · Honors · Week**, with the Honors tab leading on the reigning champion
(the one ember moment), absorbing the orphaned SeasonShelf from `/recap`, and giving
the season titles a named home. Its adversarial review left binding conditions:

- **Design-study first** (DESIGN.md pattern-vocabulary rule), and only if, after
  living with wave 1 for a while, the honors still feel homeless — wave 1's ungated
  honors block lifts onto the tab with zero rework.
- **Standings stays the year-round default tab.** Honors gets its first paint via
  the compact strip that replaced WrappedPromo — one seasonal mechanism, not a third
  computed default flip beside `liveDefaultWeekly`.
- **The tab must have a designed in-season life** (accreting shelf, designed
  "not decided yet" zero-states for champion/spoon). If the design study can't make
  week-6 Honors worth a tap, wave 1 is the permanent answer.
- Cheap validation first: there are six users — ask them what they open the app for
  before spending the 3-tab maximum. The interest ranking above is first-principles,
  not instrumented.

### Orphans to file separately

- RecapFlash/WrappedFlash: pick one season-story promoter (P3, no owner today).
- `/recap` season-qualified anchors, if split from wave 1.
- #692 scope check (finding 11).

## What was deliberately rejected

- **Rating as a standings-table column** — the canonical ADR-0032 §9 conflation;
  also the #631 no-fourth-column rule, and the 390px table already clips Total.
- **Consolidating the rating entirely to `/league`** — violates ADR-0032's surfacing
  decision and answer-first on `/stats`; the chip + always-on ladder gets the same
  legibility without the ADR amendment.
- **Sticky per-user scope memory** — hidden per-device state vs the URL-driven
  surface convention; personalization machinery for six users.
- **Calendar-driven "conversation-first" defaults** — systematizes the default
  confusion; the app feeds the chat, it doesn't host it.
